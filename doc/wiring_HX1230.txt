Подключение дисплея HX1230 к контроллерам AVR по интерфейсу SPI порождает одну проблему: в имеющихся
в продаже версиях этого дисплея по умолчанию реализован 3-проводный 9-битный вариант SPI. Контроллеры
AVR не поддерживают аппаратно этот режим.

Возможны несколько вариантов решения этой проблемы:

1. Использовать программную реализацию SPI. Это медленно и ресурсоёмко.
2. Использовать аппаратную реализацию, передавая 2 байта подряд. При этом после передачи 2-го байта
   немедленно снимать с дисплея сигнал выбора, что будет обозначать для него оборванную передачу.
   Вдвое медленнее обычной передачи.
3. Доработать дисплей, вытащив из него сигнал DC и перепаяв конфигурационные пины для включения режима
   4-Wire SPI.
4. Передавать 9-й бит отдельно, после установки сигнала CS, но до начала собственно передачи. Потребует
   дополнительных схем либо переключения "на лету" режима порта B контроллера.
5. Буферизировать 9-битные данные, передавая их через 8-битную подсистему.
